# Script Reference

## build_toc.sh

Generates `CONTEXT_TOC.md` files from `CONTEXT.yml` configuration and the front
matter of context resources.

### Usage

```bash
bin/build_toc.sh                          # Find and build all CONTEXT_TOC.md files
bin/build_toc.sh path/to/CONTEXT/         # Build one specific CONTEXT/ directory
bin/build_toc.sh path/to/CONTEXT.yml      # Build from a specific CONTEXT.yml
bin/build_toc.sh path/to/CONTEXT_TOC.md  # Build from a specific CONTEXT_TOC.md
```

### Behavior

When called without arguments, the script:

1. Finds every `CONTEXT.yml` under the current directory (excluding `.git/`)
2. Builds a skip list from all `read_only` entries across all found configs
3. For each non-skipped `CONTEXT.yml`, writes a fresh `CONTEXT_TOC.md` beside it

Each `CONTEXT_TOC.md` is overwritten completely on each run.

### Requirements

- Bash 3.2 or newer (macOS system bash is fine)
- `awk` (POSIX-compatible)
- `mktemp`
- `find`

No external dependencies. Works on macOS, Linux, and other Unix-like systems.

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | All TOCs built successfully |
| 1 | A specific file/directory argument was invalid |

## format_md.py

Formats Markdown tables to fixed-width column alignment. Useful for cleaning up
tables generated by LLMs that don't pad columns consistently.

### Usage

```bash
python3 bin/format_md.py <file.md>           # Format a file in place
python3 bin/format_md.py <file.md> -o out    # Write to a different file
cat file.md | python3 bin/format_md.py -     # Read from stdin, write to stdout
```

### What it does

Finds consecutive lines beginning with `|` (Markdown table rows), determines the
maximum content width of each column, and rewrites every row with consistent padding.
The separator row (`| --- | --- |`) is also reformatted to match. Alignment markers
in the separator (`:---`, `---:`, `:---:`) are preserved.

### Example

Input:
```
| Name | Value | Notes |
|---|---|---|
| foo | 1 | short |
| longername | 1234 | a longer note here |
```

Output:
```
| Name       | Value | Notes              |
| ---------- | ----- | ------------------ |
| foo        | 1     | short              |
| longername | 1234  | a longer note here |
```

### Requirements

- Python 3.6+
