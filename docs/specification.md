# Specification

## Overview

A context-md context node is a directory named `CONTEXT/` containing three reserved
files and any number of context documents.

```
CONTEXT/
├── CONTEXT.yml                        (required) config and metadata
├── CONTEXT_MD_SYSTEM_INSTRUCTIONS.md  (required) LLM entry point
├── CONTEXT_TOC.md                     (generated) auto-generated discovery index
└── *.md                               (optional)  context documents
```

## Reserved Files

### CONTEXT.yml

The config file for this context node. Required for `build_toc.sh` to process the
directory. Defines the node's identity (title, description) and controls what the
script indexes.

```yaml
title: Human-readable name for this context node
description: One-line summary (used in parent's Subfolders table)

depth: 1
skip_underscore: true
ignore: []
read_only: []
eager_read: []
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `title` | string | `(untitled)` | Heading in the generated CONTEXT_TOC.md |
| `description` | string | *(empty)* | Summary shown in parent TOC entries |
| `depth` | integer | `1` | Discovery depth; currently only depth=1 is implemented |
| `skip_underscore` | boolean | `true` | Skip files/folders beginning with `_` |
| `ignore` | list | `[]` | Names to exclude entirely |
| `read_only` | list | `[]` | Names to include in TOC but not rebuild (for external/symlinked resources) |
| `eager_read` | list | `[]` | Folder names the LLM should always load alongside this TOC |

### CONTEXT_MD_SYSTEM_INSTRUCTIONS.md

**The entry point for AI assistants.** This is the file that cursor rules and skills
direct agents to read at session start. It serves two purposes in one file:

1. **System explanation** — explains what context-md is and how to use it. An agent
   that has never seen context-md reads this section to understand the system. Because
   the format is consistent across all projects, an agent only needs to learn it once.

2. **Project guidance** — project-specific instructions: what this codebase does,
   which subfolders to load for which kinds of work, key relationships, caveats.

The file has a standard structure (copy from `templates/CONTEXT_MD_SYSTEM_INSTRUCTIONS.md`):

```markdown
# Context Instructions

## How This System Works
[standard boilerplate explaining context-md — do not remove]

## About This Project
[your project-specific guidance here]
```

Human-edited. Never modified by the script. Never appears in the Files table of any
generated TOC.

### CONTEXT_TOC.md

Fully generated by `build_toc.sh`. Do not edit by hand. See [TOC Format](toc-format.md)
for the output structure.

## Context Documents

Any `.md` file in a `CONTEXT/` directory (other than reserved files) is a context
document. Each should have YAML front matter:

```markdown
---
title: Document Title
description: One-line summary for the TOC entry
---

Content...
```

- `title` — used as the link label in the TOC Files table
- `description` — the description column in the TOC Files table
- If either is missing, the script falls back to the filename (without `.md`)

## Discovery Rules

When `build_toc.sh` processes a `CONTEXT.yml`, it builds the Subfolders and Files
tables by scanning two locations:

**Subfolders table** — populated from:
1. Immediate subdirectories of `CONTEXT/` that contain a `CONTEXT.yml`
2. Directories adjacent to the project root that contain a `CONTEXT/CONTEXT.yml`

**Files table** — populated from:
- `.md` files directly in `CONTEXT/` (reserved files are always excluded)

**Filtering:**
- `skip_underscore: true` — silently skips names beginning with `_`
- `ignore` list — silently skips listed names
- `read_only` list — includes the name in the TOC (marked *read-only*) but skips
  rebuilding its own `CONTEXT_TOC.md` in auto-discovery mode

## Agent Entry Point Wiring

The `CONTEXT_MD_SYSTEM_INSTRUCTIONS.md` file is only useful if agents know to read it.
Wire this up via one of:

**Skill** — symlink `skills/context-md.md` from this repo into your project's skills
directory. The skill's description tells the agent to read the file at session start.

**Cursor rule** — copy `templates/cursor-rule.mdc` into `.cursor/rules/`. The
`alwaysApply: true` frontmatter ensures it is included in every session.

## Portability via Symlinks

Any `CONTEXT/` subfolder can be a symlink to an external directory. Discovery follows
symlinks normally. Add its name to `read_only` to prevent `build_toc.sh` from
modifying files inside the linked resource.
