# Context Instructions

## How This System Works

This project uses **context-md** to organize background knowledge for AI assistants.

```
CONTEXT/
├── CONTEXT_MD_SYSTEM_INSTRUCTIONS.md  ← this file (entry point, read first)
├── CONTEXT_TOC.md                     ← auto-generated discovery index
├── CONTEXT.yml                        ← config (not for LLM consumption)
└── *.md                               ← context documents
```

**Read this file once per session.** If you encounter another project that also uses
context-md, you do not need to read its system instructions again — the system works
the same way everywhere.

### Using Context Efficiently

After reading this file:

1. **Read `CONTEXT/CONTEXT_TOC.md`** — the full index of available context with
   one-line descriptions for every subfolder and file.
2. **Scan descriptions** — identify which resources are relevant to your current task.
3. **Fetch only what you need** — do not load all context upfront.
4. **For subfolders**, read that folder's `CONTEXT_TOC.md` before its individual files.
5. **Eager-read folders** — if the TOC lists an "Always Load" section, load those
   folders immediately.

## About This Project

This is the **context-md** project itself — the repository that defines the standard
and ships `bin/build_toc.sh`.

If you are helping develop or extend this project:

- **`bin/build_toc.sh`** is the core tool. It reads `CONTEXT.yml` + finds `.md` files,
  then writes a fully generated `CONTEXT_TOC.md`. The script must stay bash 3.2
  compatible (macOS system bash). Do not use associative arrays (`declare -A`),
  namerefs (`local -n`), or other bash 4+ features.
- **`bin/format_md.py`** is a helper for formatting markdown tables. Secondary concern.
- **`CONTEXT_TOC.md` files are never hand-edited.** They are always regenerated by the script.
- **`CONTEXT.yml`** is the human-facing config per context node.
- The `examples/` directory shows a working example project structure with live
  generated TOC files.
- Docs live in `docs/` and are served via GitHub Pages (docsify, buble.css theme) at
  https://cart0113.github.io/context-md/
